# PRD — Nueva Funcionalidad (Feature)
Este PRD se usa para cualquier nueva funcionalidad del proyecto 100-Tráfico.

## 1. Problema a resolver
Describe en 1-2 frases el problema específico que quieres resolver.

## 2. Usuario principal
¿Quién usa esta funcionalidad dentro del sistema?

## 3. Proceso actual (sin la feature)
Explica cómo se está resolviendo hoy ese problema.

## 4. Flujo del usuario (5–7 pasos)
Describe cómo debería interactuar con la nueva función.

## 5. Datos necesarios
¿Qué debe almacenar, actualizar o leer la app?

## 6. Reglas y validaciones
Condiciones, restricciones, límites, estados.

## 7. Integraciones
¿Habla con API? ¿Scheduler? ¿Workers? ¿Bot? ¿Frontend?

## 8. MVP
¿Qué sí incluye?
¿Qué NO incluye?

## 9. Riesgos
Posibles fallas, cuellos de botella o dependencias.

## 10. Confirmación
Cuando estés listo: “Genera el PRD final”.
